# Project Working Rules

## Iteration Review Flow

- After each code-editing turn, keep changes uncommitted and stop for review unless the user explicitly asks to continue.
- Do not stage, commit, push, or revert changes unless the user explicitly requests that action.
- Keep each editing turn narrowly scoped so the Codex app review pane can show useful Last turn changes.
- In final responses after edits, remind the user to use the Codex app review pane to audit or revert the latest changes.
- Prefer tracked files in this Git repository for code changes; mention when a requested edit affects files outside Git tracking.
